# 架构说明

## 分层

- Renderer：React + TypeScript，负责七阶段工作流、语义中心和状态呈现。
- Preload：将产品级 `window.workbench` 接口映射到白名单 IPC。
- Main：Electron 窗口、文件选择、Finder、自动更新与 IPC 注册。
- Workspace Service：任务、文件、提示词、执行回执和权威语义的本地读写。

## 数据流

`外部资料 -> inbox -> raw -> AI 调度 -> 四件套/回执 -> 人工验证 -> 输出 -> AI评测 -> 语义候选`

资料不完整时仍生成当前可用报告；缺口进入验证清单并限制结论边界。权威语义层是跨任务资产，AI 只能提出候选，人工确认后才成为正式定义。

## 可升级性

Renderer 与本地服务通过稳定接口解耦。后续可加入模型适配器、语义版本治理、可配置评测题库和跨平台构建，而不改变用户工作区格式。
